# 场景分析 - 孵化器管理

## 场景概述

| 项目 | 说明 |
|------|------|
| 场景名称 | 孵化器管理（A6 奇岱松校友中心试点） |
| 涉及部门 | 孵化器运营（赵婧总/公台红） |
| 核心用户 | 孵化器运营人员（日常）、赵婧总（管理）、领导层（汇报） |
| 核心痛点 | ① 70 家孵化企业和 17000+ 园区企业的合作靠人脑匹配 ② 运营指标无量化手段 ③ 企业信息分散在多维表格，不结构化 |

## 假设决策

- 运营监控：**先做软件可达的**（会议室预约 + 访客记录），人流/门禁硬件延后
- 订单匹配：**核心 AI 亮点**，用对话式交互 + 可视化呈现
- 成长评估：**延后到二期**，没有方法论基础
- 数据源：孵化企业靠 **解析 BP** 补齐画像，不依赖企业主动填写

## 业务流程

```
日常运营监控 → 订单/资源智能匹配 → (成长评估-延后)
  (运营指标)      (AI 核心亮点)
```

### 阶段 1：日常运营监控

- **输入**：
  - 会议室预约数据（校友软件 → 静态报表拉取）
  - 访客登记数据（飞书多维表格，实时）
  - 孵化企业基本信息（多维表格 + BP 解析）

- **操作**：
  1. 定期拉取会议室预约数据和访客登记数据
  2. 按企业维度聚合：每家企业的会议频次、访客量、访客类型（商务/招聘/投资机构）
  3. AI 分析活跃度指标，识别高活跃企业：
     - 高频会议 → 可能在融资或扩张
     - 大量招聘访客 → 可能在快速增长
     - 投资机构来访 → 可能在融资
  4. 生成高活跃企业报告，推送给运营人员
  5. 运营看板展示：企业活跃度排名、异常预警（如某企业突然零活动）

- **输出**：
  - 运营监控看板（企业活跃度、空间使用率）
  - 高活跃企业报告
  - 异常预警（可选）

- **痛点**：
  - 现在：运营人员靠感觉判断哪个企业发展好
  - 不知道哪些企业在扩张、哪些在萎缩
  - 领导要数据汇报时只能手动统计

- **对应功能**：
  - 会议室预约系统接入（P1，0.2）
  - 访客登记系统接入（P2，0.2）—— 孵化器场景
  - 高活跃企业报告 Agent（P1，0.2）
  - 日常运营监控看板（P0，0.1）—— 基础版
  - ~~奇岱松人流趋势监控~~（延后，需硬件）
  - ~~打卡数据系统~~（延后，外部系统）

### 阶段 2：订单/资源智能匹配

- **输入**：
  - 孵化企业画像（BP 解析 + 结构化信息 + 历史订单）
  - 漕河泾园区企业数据库（17000+ 企业）
  - 用户输入的匹配查询（对话式）

- **操作**：
  
  **场景 A：外部需求 → 匹配孵化企业**
  1. 运营人员（或领导走访回来）输入一个需求："A 企业有个自动洗车项目，我们孵化的企业有谁能参与？"
  2. AI 拆解需求：自动洗车 → 机械臂 / 传感器 / 软件控制 / 清洗材料 ...
  3. AI 在孵化企业画像中匹配：哪些企业的产品/技术/能力与拆解后的子任务相关
  4. 输出匹配列表：企业名 + 匹配原因 + 匹配度 + 企业活跃度
  5. 优先推荐：A6 注册主体 > 上海主体 > 外地主体

  **场景 B：孵化企业变化 → 推荐园区合作方**
  1. 系统检测到某孵化企业有变化（新入驻 / 完成融资 / 发布新产品 / BP 更新）
  2. AI 分析该企业的能力和需求
  3. 在 17000+ 园区企业库中匹配可能的合作方（上下游 / 客户 / 供应商）
  4. 输出推荐列表 + 合作可能性分析
  5. 推送给运营人员审核

  **场景 C：关系网可视化**
  1. 选中一家孵化企业
  2. 以可交互网状图展示该企业与园区企业的产业链关系
  3. 支持点击节点查看详情、展开二级关系
  4. 用于领导汇报和资源对接会议

- **输出**：
  - 匹配推荐列表（对话式输出）
  - 关系网可视化图（前端页面）
  - 合作可能性分析报告

- **痛点**：
  - 现在：运营人员靠个人经验匹配，熟悉的企业优先，不熟的就 miss 了
  - 领导想看全景但没有工具
  - 不知道孵化企业还能做什么、园区有什么资源可以对接

- **对应功能**：
  - 订单匹配 Agent（P0，0.1）
  - 根据园区需求分析孵化企业匹配（P0，0.1）
  - 孵化企业新入/变更分析园区企业配套匹配（P0，0.1）
  - 关系网可视化/页面分析（P1，0.2）
  - 漕河泾企业数据库接入（P0，0.1）
  - 开发区及孵化器企业数据库建设（P0，0.1）

### 阶段 3：成长评估（延后到二期）

> 赵婧总想要一个"成长评估模型"：对不同赛道的孵化企业，建立成长基准线（第几个月应该有产品、第几年应该有收入），然后对比实际情况做预警。
>
> **延后原因**：
> 1. 没有现成的方法论——赵婧总自己也说不清楚标准是什么
> 2. 数据量不足——70 家企业、10 个案例，无法训练或校准模型
> 3. 需要先积累 1-2 个季度的运营数据才有意义
>
> **建议**：一期先积累数据（走访记录 + 运营指标 + 订单匹配结果），二期基于积累的数据设计评估体系。
