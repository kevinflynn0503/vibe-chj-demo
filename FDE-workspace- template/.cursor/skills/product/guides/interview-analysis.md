# 访谈纪要分析方法

## 纪要类型

| 类型 | 特征 | 处理方式 |
|------|------|---------|
| 人工笔记 | 要点式，可能不完整 | 重点关注笔记者标注的重要内容 |
| 录音转写 | 完整但冗余 | 按人物过滤，提取关键诉求 |
| 飞书妙记 | 自动标注时间点 | 关注高频出现的关键词 |

## 分析步骤

### 1. 通读一遍，标注人物

- 识别每个发言人的角色和部门
- 建立"谁关心什么"的映射

### 2. 按人物提取诉求

- 每个人说了什么需求
- 每个人的痛点是什么
- 每个人的优先级偏好

### 3. 识别信号

| 信号 | 含义 | 处理 |
|------|------|------|
| "最头疼的是..." | 高优先级痛点 | 标注 P0 候选 |
| "顺便说一下..." | 隐藏需求 | 不要忽略 |
| "以前试过..." | 有历史经验 | 记录失败原因 |
| "领导说要..." | 政治需求 | 标注为刚需 |
| "能不能..." | 探索性需求 | 需确认真实意图 |

### 4. 注意事项

- 不要过度解读：纪要是二手信息，有些语气语境在文字中丢失
- 标注不确定的：宁可多问 FDE 一句，不要自己脑补
- 交叉验证：不同人提到同一件事，优先级更高
