# Phase 2：需求结构化

> 输入：澄清后的需求 + FDE 补充信息
> 输出：用户需求池终版 + 场景分析 + 数据需求清单
> 时间：Day 3-4

## 执行步骤

### 1. 更新需求池

- 标记已确认 / 已拒绝 / 优先级调整
- 补充 FDE 带回的新信息

### 2. 按场景分组

将零散需求组织为连贯的业务场景：

```
用户需求池（零散的点）         →        场景化结构（连贯的面）
┌─────────────────┐                    ┌────────────────────────┐
│ UR-001 背调报告  │                    │ 场景：企业走访           │
│ UR-002 妙记抓取  │    AI 组织整理      │ ├── 拜访准备（背调+提纲）│
│ UR-003 企业画像  │ ─────────→         │ ├── 拜访记录（妙记+提取）│
│ ...              │                    │ ├── 需求处理（匹配+分发）│
└─────────────────┘                    │ └── 管理看板（统计+报表）│
                                       └────────────────────────┘
```

### 3. 梳理业务流程

每个场景用模板 `templates/scenario-analysis.md`：
- 阶段划分
- 每个阶段的输入/输出
- 每个阶段的痛点和对应功能

### 4. 识别共性

- 跨场景共用数据（企业画像库）
- 跨场景共用能力（飞书集成、报告生成）

### 5. 梳理数据依赖

| 功能 | 需要什么数据 | 数据在哪 | 什么格式 | 谁维护 |
|------|------------|---------|---------|--------|

### 6. 输出数据需求清单

驱动 FDE 去收集数据样本、了解现有系统。

## HITL 交互

AI 输出数据需求清单 → FDE 去客户那收集数据样本 → 带回来 → 进入 Phase 3。
