## 一、架构澄清：App 与小北的关系

### 小北是什么

小北是一个 AI 驾驶舱平台。App 作为 iframe 嵌入小北的右侧面板。关键机制：

```
┌─────────────────────────────────────────────────────┐
│                     小北平台                          │
│                                                      │
│  ┌──────────────┐    ┌────────────────────────────┐  │
│  │              │    │                            │  │
│  │   左侧       │    │   右侧面板（iframe）        │  │
│  │   对话区      │    │                            │  │
│  │              │    │   ┌────────────────────┐   │  │
│  │   用户和 AI   │    │   │                    │   │  │
│  │   在这里对话  │    │   │   漕河泾 App        │   │  │
│  │              │    │   │  （我们要做的）       │   │  │
│  │   Agent 在   │    │   │                    │   │  │
│  │   这里执行    │    │   │                    │   │  │
│  │              │    │   └────────────────────┘   │  │
│  └──────────────┘    └────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 关键原则

1. **App 只做 UI + 数据展示**，不做 AI 推理
2. **需要 AI 的地方 → 调 `sendChat`** → 小北左侧对话区触发 Agent → Agent 写 Supabase → App 通过 Realtime 接收结果
3. **App 是一个整体嵌入小北**，内部可以有自己的导航（类似浏览器标签页）
4. **App 不需要外壳导航**（小北已经有），但内部功能多时需要自己的导航方式

### 正确的技术架构

```
用户操作 App UI
       │
       ├─── 普通操作（查看/筛选/编辑）→ App 内部处理
       │
       └─── AI 操作（生成背调/政策筛选/订单匹配）
                │
                ▼
            sendChat(prompt)
                │
                ▼
            小北 Agent 执行
                │
                ▼
            Agent 调用 supabase_tool 写入数据库
                │
                ▼
            App 通过 Supabase Realtime 收到更新
                │
                ▼
            UI 实时刷新展示结果
```

## 二、App 内部导航设计

App 功能较多（三个场景 + 公共企业画像），需要内部导航。

### 方案：应用内标签页（推荐）

类似浏览器标签页，但更轻量：

```
┌──────────────────────────────────────────────────────┐
│ 漕河泾智能驾驶舱                                       │
│                                                       │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌──────────┐   │
│  │客户拜访 ×│ │政策服务  │ │孵化器   │ │ +        │   │
│  └─────────┘ └─────────┘ └─────────┘ └──────────┘   │
│ ─────────────────────────────────────────────────── │
│                                                       │
│   当前标签页的内容区域                                   │
│                                                       │
│                                                       │
│                                                       │
└──────────────────────────────────────────────────────┘
```

**设计要点：**
- 顶部一行标签，不占太多空间
- 可以关闭标签（×）
- 可以新开标签（+）
- 默认打开「客户拜访」标签
- 在企业画像等子页面也可以作为新标签打开（类似浏览器新 Tab）
- 嵌入小北时这个标签栏保留（它是 App 内部导航，不是平台导航）

## 九、总结：核心纠偏

| 维度 | 之前的理解 | 正确的理解 |
|------|-----------|-----------|
| 架构 | 三个App分开 or 合并成一个 | 一个App嵌入小北，内部用标签页切换三个场景 |
| AI | App内部实现AI功能 | AI统一走sendChat→小北Agent→Realtime回来 |
| 首页 | 全局KPI大屏 | 每个场景Tab有自己的首页（企业列表/工作台/运营概览） |
| 交互 | 展示数据 | 驱动工作流（每个页面都有"下一步做什么"） |
| 飞书 | 没有体现 | 贯穿全流程（妙记/日历/任务/文档的图标+状态+链接） |
| 风格 | 深色科技Dashboard | 飞书风格：浅色、紧凑、操作导向 |
| 企业画像 | 简单详情页 | 企查查风格：多数据源、高密度、按场景显示Tab |
| 导航 | 顶部全局导航栏 | App内标签页（类浏览器Tab），嵌入小北时保留 |
