## 🚀 快速开始

### 新成员入门（推荐阅读顺序）

1. **第1天**：阅读 [UI规范对照表](./UI规范对照表.md)（15分钟）
   - 了解基本规范
   - 记住常用类名
   
2. **第1周**：阅读 [UI规范快速实施指南](./UI规范快速实施指南.md)（30分钟）
   - 学习如何创建组件
   - 完成配置更新
   - 试点重构一个小页面

3. **第2周**：阅读 [UI规范统一深度分析](./UI规范统一深度分析.md)（1小时）
   - 理解规范背后的设计思想
   - 了解完整的实施计划

### 老成员规范化改造

直接从 [UI规范快速实施指南](./UI规范快速实施指南.md) 开始，按照 Step 1-4 执行。

**推荐阅读**：

  - 第5章：实施路线图

---

### 👨‍💻 前端开发工程师
**关注点**：怎么做、具体代码、快速上手

**推荐阅读**：
1. [UI规范快速实施指南](./UI规范快速实施指南.md)（必读）
2. [UI规范对照表](./UI规范对照表.md)（日常查询）
3. [UI规范统一深度分析](./UI规范统一深度分析.md) - 第4章（组件规范）

**日常工作流**：
- 开发新功能时：查阅 [对照表](./UI规范对照表.md)
- 重构旧代码时：参考 [实施指南](./UI规范快速实施指南.md) 的批量替换清单
- 遇到特殊场景：查阅 [深度分析](./UI规范统一深度分析.md) 的组件扩展方案

---

### 🎨 UI/UX 设计师
**关注点**：设计规范、视觉一致性、Design Tokens

**推荐阅读**：
- [UI规范统一深度分析](./UI规范统一深度分析.md)
  - 第2章：核心问题分析（了解现状）
  - 第3章：Design Token 体系设计（了解设计系统）

**协作建议**：
- 新增设计元素前，先查看 Design Token 是否已覆盖
- 设计交付时，标注使用的 Token（如 `text-lg`, `rounded-lg`）

---

### 🔍 Code Reviewer
**关注点**：快速识别不规范代码、审查标准

**推荐阅读**：
- [UI规范对照表](./UI规范对照表.md)（快速对照）
- [UI规范快速实施指南](./UI规范快速实施指南.md) - ✅ 自检清单

**审查时重点检查**：
- [ ] 无 `text-[10px]`, `text-[11px]` 等任意值
- [ ] 无 `rounded-[10px]` 等任意圆角值
- [ ] 卡片使用 `<Card*>` 组件而非手写样式
- [ ] 标签使用 `<Tag>` 组件
- [ ] 颜色使用 `text-text-*` 或 `text-brand`
