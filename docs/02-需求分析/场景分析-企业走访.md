# 场景分析 - 企业走访

## 场景概述

| 项目 | 说明 |
|------|------|
| 场景名称 | 企业走访（客户拜访） |
| 涉及部门 | 产促部（蔡建总）、科创中心、园发中心、事业部租赁团队 |
| 核心用户 | 招商经理（一线）、赛道组长（配置）、部门负责人（管理） |
| 核心痛点 | ① 拜访前准备靠人工，新人不会谈 ② 拜访后信息不沉淀，跨部门不互通 ③ 管理者不知道下面人干了啥 |

## 假设决策

- 烽火 CRM：**替代**，不回填，我们的平台成为唯一的走访信息沉淀入口
- 企业画像：**共用一套**，走访/政策/孵化共享同一个企业画像页
- 数据更新：AI 提取走访记录后 **人工确认再入库**
- 政策联动：走访时 **同时带政策部门的必问问题**

## 业务流程

```
拜访准备 → 拜访实施 → 拜访复盘 → 需求闭环
 (AI重点)    (妙记录音)   (AI重点)    (一期简版)
```

### 阶段 1：拜访准备

- **输入**：
  - 要走访的企业名称（招商经理自行决定或 leader 安排）
  - 企业数据库中的已有信息
  - 历史走访记录（如有）
  - 赛道知识库（3+3 必问问题）
  - 政策部门嵌入的话术问题（如有匹配的政策）

- **操作**：
  1. 招商经理在平台上选择要走访的企业
  2. 系统自动拉取企业画像（工商/股权/融资/知识产权/动态）
  3. 系统查找历史走访记录并汇总
  4. AI 生成背调报告（飞书文档），包含：
     - 企业基本面（8 个模块：工商/股权/融资/团队/产品/动态/产业链/潜在需求）
     - 谈话提纲（基于赛道 3+3 + 企业特点定制的合作方向建议）
     - 政策触达问题（如果该企业匹配了某个政策，嵌入科创的必问问题）
  5. AI 生成沟通清单，包含背调中的关键洞察

- **输出**：
  - 背调报告（飞书文档，可打印带去）
  - 沟通清单（谈话提纲 + 必问问题）

- **痛点**：
  - 现在：人工查企查查 + 人工写背调 → 2 小时/家
  - 新兵不知道问什么，老法师经验在脑子里不沉淀
  - 科创部门不知道招商出去时有没有帮他们问政策问题

- **对应功能**：
  - 企业画像管理（P0，0.1）
  - 背调报告生成（P0，0.1）—— 工商信息/股权/融资/团队/产品/动态/产业链/潜在需求/潜在合作/历史走访汇总
  - 沟通清单生成（P0，0.1）—— 清单生成 + 关键洞察
  - 走访历史管理（P0，0.1）
  - 赛道关键问题配置（P0，0.2）
  - 走访必问问题生成（政策服务 app，P0，0.1）

---

### 阶段 2：拜访实施

- **输入**：
  - 背调报告 + 沟通清单
  - 飞书日历中的拜访日程

- **操作**：
  1. 招商经理在飞书日历中创建日程（标记为拜访 + 关联企业）
  2. 拜访时开启飞书妙记录音
  3. 按照沟通清单进行谈话
  4. 拜访结束，关闭妙记

- **输出**：
  - 飞书妙记录音 + 转录文本

- **痛点**：
  - 部分员工抵触录音（个人隐私/关系维护）
  - 方言识别不准
  - ~~实时策略生成~~（已取消）

- **对应功能**：
  - 飞书日历接入（P0，0.1）
  - 飞书妙记接入（P0，0.1）
  - ⚠️ 无平台功能——保持用户现有习惯，不增加操作步骤

---

### 阶段 3：拜访复盘

- **输入**：
  - 飞书妙记（录音 + 转录）
  - 飞书日历（拜访日程）
  - 当前企业画像

- **操作**：
  1. 系统自动将妙记与日历中的拜访日程关联（匹配时间+参会人），关联到对应企业
  2. 如自动关联失败，推送消息让招商经理手动关联
  3. AI 分析妙记内容，自动提取：
     - 企业最新情况（营收/人数/研发投入等被提及的数据）
     - 企业诉求（服务需求/合作需求/政策需求）
     - 合作意向和下一步计划
  4. 以结构化方式展示给招商经理确认
  5. 招商经理调整/补充（人工补充未录到的观察、约定事项）
  6. 确认后写入企业画像（AI 丰富标签：产业链位置、发展阶段等）
  7. AI 检测本次拜访是否覆盖了预设的赛道关键问题，生成覆盖度报告
  8. 如果走访信息满足政策条件，自动推送给科创部门

- **输出**：
  - 走访记录（结构化，已确认）
  - 企业画像更新
  - 关键问题覆盖度报告（给 leader）
  - 政策线索推送（给科创）

- **痛点**：
  - 现在：走访后要手动填烽火 CRM，大多数人不填
  - 同一企业被多个部门走访，信息不互通
  - 管理者无法评估走访质量

- **对应功能**：
  - 妙记自动关联（P0，0.2）
  - 走访记录生成（P0，0.2）
  - 走访记录调整（P1，0.2）
  - 关键问题检测（P1，0.2）
  - 跨部门信息同步（P1，0.3）
  - 企业数据库自动更新（隐含，P1）

---

### 阶段 4：需求闭环（一期简版）

- **输入**：
  - 走访记录中提取的企业诉求
  - 部门职责对照表

- **操作**：
  1. AI 从走访记录中提取企业诉求，整理成需求清单
  2. 每条需求标注：需求内容 / 类型（服务/合作/政策） / 来源走访记录
  3. 招商经理确认需求清单
  4. 确认后自动生成飞书任务，分发给对应部门

- **输出**：
  - 企业需求清单
  - 飞书任务（自动 @ 对应部门）

- **痛点**：
  - 现在：企业诉求写在各部门自己的多维表格里，没人跟进
  - 跨部门的需求不知道该找谁

- **对应功能**：
  - 需求清单整理（P1，0.3）
  - 任务自动创建及分发（P2，0.3）
  - ~~服务匹配推荐~~（一期不做 AI 自动匹配，人工分配）
  - ~~企业合作推荐~~（一期不做）
  - ~~企业政策推荐~~（一期不做，由政策场景覆盖）
