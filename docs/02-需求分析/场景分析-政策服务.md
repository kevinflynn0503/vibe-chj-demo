# 场景分析 - 政策服务

## 场景概述

| 项目 | 说明 |
|------|------|
| 场景名称 | 政策服务（一期聚焦高新技术企业认定） |
| 涉及部门 | 科创中心（薛科总）、产促部（走访协同） |
| 核心用户 | 项目经理（科创一线）、薛科总（管理）、招商经理（走访协同） |
| 核心痛点 | ① 不知道 17000 家企业中谁能报高新 ② 走访信息回不来 ③ 人工审核材料耗时 |

## 假设决策

- 一期聚焦：**仅做高新技术企业认定**，跑通后复制到专精特新等其他政策
- 政策 × 走访：**合并**，走访时同时带政策问题，走访信息自动推送给科创
- 申报材料填写：**延后到二期**，一期只做诊断不做代写
- 面向对象：**仅内部**，不对外开放给企业

## 业务流程

```
潜在企业筛选 → 政策触达 → 申报诊断 → 结果跟踪
  (AI 核心)     (任务分发)   (AI 辅助)   (延后/人工)
```

### 阶段 1：潜在企业筛选

- **输入**：
  - 企业数据库（17000+ 企业的工商/知识产权/年报数据）
  - 高新技术企业认定管理办法（硬性条件 rubrics）
  - 外部公开数据（企查查/天眼查/公开新闻）
  - 走访场景回流的企业信息（妙记提取的营收、研发人员等）

- **操作**：
  1. 系统管理员上传政策材料（认定办法 + 所需材料清单）
  2. AI 自动拆解硬性条件，生成初筛 rubrics（如：成立≥1年、拥有自主知识产权、技术领域属于国家重点支持等）
  3. 对企业数据库中的企业批量跑初筛：
     - 数据充分的字段 → 自动判断（通过/不通过）
     - 数据不足的字段 → 标记"待确认"
  4. deep research 智能体补齐：对"待确认"的企业，从公开数据搜集补充信息
  5. 输出筛选结果并按匹配程度分级（A/B/C）：
     - A 级：硬性条件全部满足，高概率可申报
     - B 级：大部分满足，需进一步确认 1-2 项
     - C 级：部分满足，需深入了解
  6. 生成潜在企业名单，推送给科创部门

- **输出**：
  - 潜在企业名单（含分级 A/B/C）
  - 每家企业的初筛报告（哪些条件满足、哪些待确认、缺什么数据）

- **痛点**：
  - 现在：薛科总靠人工从名单里找，或者问田永俊要清单再一个个查
  - 不知道企业够不够条件，出去走访才发现白跑
  - 闵行用外部智能体做，徐汇用我们做，要对比精准度

- **对应功能**：
  - 政策材料上传（P0，0.1）
  - 潜在企业初筛 rubrics 生成（P0，0.1）
  - 潜在企业初筛智能体（P0，0.1）
  - deep research 智能体（P0，0.1）
  - 潜在企业分级 rubrics 生成（P0，0.1）
  - 潜在企业分级智能体（P0，0.1）
  - 企业管理大盘（P0，demo）

### 阶段 2：政策触达

- **输入**：
  - 潜在企业名单（含分级）
  - 企业对应的客户经理 / 项目经理
  - 走访必问话术（科创 7 条 + 赛道 3+3）

- **操作**：
  1. 系统根据企业分级和项目经理负责范围，自动生成走访任务
  2. 通过飞书任务分发给项目经理（A 级：3 天内走访 / B 级：1 周内 / C 级：2 周内）
  3. 系统为每次走访生成走访问题（基于初筛报告中"待确认"的项 + 政策必问话术）
  4. 项目经理走访企业，开妙记录音
  5. 走访后，AI 从妙记中提取关键信息（复用走访场景能力）
  6. 提取的信息自动回流到企业画像，并更新该企业的政策匹配状态
  7. 跟踪触达进度：已分发 / 已走访 / 有意愿 / 无意愿 / 待跟进

- **输出**：
  - 走访任务（飞书任务）
  - 走访记录（结构化，含政策相关字段）
  - 企业参与情况汇总
  - 工作进度看板数据

- **痛点**：
  - 现在：科创部门自己不走访企业，依赖招商经理，但招商经理不帮他们问
  - 走了也不知道结果，信息回不来

- **对应功能**：
  - 政策触达任务分发功能（P0，0.1）
  - 走访必问问题生成（P0，0.1）—— 与走访场景共用
  - 政策触达问题生成智能体（P1，0.2）
  - 飞书纪要获取解析（P0，0.1）—— 与走访场景共用
  - 政策触达总结智能体（P1，0.2）
  - 工作进度看板（P0，demo）

### 阶段 3：申报诊断（前置审核）

- **输入**：
  - 企业确认有意愿申报
  - 企业提供的补充材料（财务数据/产品介绍/研发制度等）
  - 政策申报审核 rubrics

- **操作**：
  1. 企业提供材料（通过项目经理上传，或走访中口头提供被妙记记录）
  2. AI 根据审核 rubrics 逐项评判：
     - 硬性条件（知识产权数量、研发费用占比、高新收入占比等）→ 量化判断
     - 软性条件（技术先进程度、核心支持作用等）→ AI 辅助评分 + 标注置信度
     - 数据不足的项 → 标记"待补充"而非编造
  3. 生成诊断报告：得分/建议/缺失项/风险提示
  4. 项目经理审核报告，与企业沟通补充材料
  5. 对于税务风险（一票否决项），特别标注提醒

- **输出**：
  - 申报诊断报告（逐项评分 + 综合建议）
  - 待补充材料清单

- **痛点**：
  - 现在：靠薛科总团队人工一份份看材料
  - 定性评分主观性大（专家一般往高打）
  - 企业财务数据获取困难（非公开，需企业主动提供）

- **对应功能**：
  - 材料上传功能（P0，demo）
  - 政策申报审核 rubrics 生成（P1，0.2）
  - 政策申报审核智能体（P1，0.2）—— 先做硬性条件判断，软性条件延后
  - ~~申报材料填写智能体~~（延后到二期）
  - ~~申报材料模板准备~~（延后到二期）

### 阶段 4：结果跟踪（一期简化/人工）

- **操作**：

  3. 政策统计看板自动更新

- **输出**：
  - 获批企业名单
  - 政策统计看板

- **对应功能**：
